# Codex Workspace Files

## Purpose

This folder gives Codex immediate project context so future tasks start faster and with fewer mistakes.

## Files

- `PROJECT_OVERVIEW.md`: product and subsystem summary.
- `ARCHITECTURE_MAP.md`: service graph, control flows, and critical modules.
- `DEV_WORKFLOW.md`: setup/test/runbook commands.
- `CHANGE_CHECKLIST.md`: quality and safety gates for code changes.
- `TASK_TEMPLATES.md`: reusable prompts for common development tasks.

## Suggested Usage

When starting a new task, read:

1. `../AGENTS.md`
2. `PROJECT_OVERVIEW.md`
3. `ARCHITECTURE_MAP.md`
   Then use `DEV_WORKFLOW.md` and `CHANGE_CHECKLIST.md` during implementation.
